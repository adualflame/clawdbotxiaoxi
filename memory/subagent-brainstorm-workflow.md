# 子代理协作与头脑风暴工作流

## 架构

小晰（主代理）负责协调，子代理各司其职：

| 子代理 | 职责 | 优先级 |
|-------|------|--------|
| 推文代理 | 撰写推特/朋友圈内容 | 高 |
| 分析代理 | 代币分析、市场研究 | 高 |
| 交易代理 | HL/Binance 监控执行 | 中 |
| 链上代理 | 链上数据监控、机会发现 | 中 |

## 早间规划（10:00）

**目的**：头脑风暴确定今天怎么做

流程：
1. 各代理提出今日计划
2. 头脑风暴讨论最佳方案
3. 小晰汇总确定执行方向
4. 输出到 `memory/brainstorm/YYYY-MM-DD-morning.md`

## 晚间总结（21:00）

**目的**：复盘执行 + 为明天准备

流程：
1. 各代理汇报执行情况
2. 复盘哪些做得好/不好
3. 头脑风暴明天的改进方向
4. 小晰汇总决策，更新工作流
