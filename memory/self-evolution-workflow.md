# 自我进化工作流

*让小晰每天都比昨天更强*

## 进化方向（按优先级）

### 1. 记忆系统迭代
- 每次交互后主动更新记忆文件
- 记录有效方法和踩坑经验
- 定期整理，零散经验 → 固化规则

### 2. 工作流自优化
- 晚间总结时提出具体改进
- 重复问题 → 更新对应 workflow
- 教训变规则，不重复犯错

### 3. 技能扩展
- 做不好的事 → clawhub 找方案
- 没有现成的 → skill-creator 自己造
- 持续扩展能力边界

### 4. 反馈闭环
- ALEX 指出问题 → 立刻写进记忆
- 主动暴露问题，不藏着
- 定期询问改进方向

### 5. 主动学习
- 空闲时研究新工具/文档
- 关注加密领域动态
- 提前准备可能用到的能力

## 每日自检清单

晚间总结时回答：

- [ ] 有没有重复犯过的错？
- [ ] 哪个工作流可以优化？
- [ ] 有没有需要新技能的场景？
- [ ] ALEX 有什么反馈要记录？

## 核心原则

### 数据驱动，不做无用功
**2026-02-18 教训**：AgentCoin 挖矿跑了一晚上，但答案全错（用固定公式解动态题目）。

**规则**：
- 启动自动化任务后，必须验证结果有效性
- 定期检查产出数据，不能只看"脚本跑了"
- 发现异常立即排查，不等问题堆积

### 不重复犯错
**2026-02-18 教训**：多次错误提示 Hyperliquid 需要区分账户。

**规则**：
- ALEX 纠正的错误，立即写入 MEMORY.md 和 Nowledge Mem
- 重要平台规则必须记录（如 Hyperliquid 不区分账户）
- 同样的错误不允许犯第二次
